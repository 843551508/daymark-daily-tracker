# 参与贡献

## 开发原则

- 保持无构建步骤，浏览器应能直接加载源码。
- 保持本地优先，不引入必须联网的核心依赖。
- 不提交数据库、备份或任何真实个人数据。
- 修改共享状态结构时更新 `version`、迁移逻辑和文档。
- 新功能需要覆盖空状态、错误状态和手机布局。

## 本地运行

```bash
php -S 127.0.0.1:8787
```

若 SQLite3 未默认启用：

```bash
php -d extension=sqlite3 -S 127.0.0.1:8787
```

## 检查

```bash
node --check assets/app.js
php -l api.php
```

还应在 `test.html` 运行 API 自检，并验证桌面和 390px 手机布局。

## Pull Request

PR 应说明：

- 修改内容与原因
- 用户影响
- 数据结构或兼容性变化
- 已运行的检查
- 涉及界面时附截图
